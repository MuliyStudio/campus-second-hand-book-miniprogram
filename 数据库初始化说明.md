# 数据库初始化说明

## 问题背景
发布书籍功能出现"图片审核服务异常"错误，主要原因是：
1. 缺少必要的数据库集合（books、audit_logs）
2. 图片审核云函数超时
3. 网络请求超时处理不完善
4. 小程序端上传图片时传入了无效的临时文件路径

## 解决方案
1. **初始化数据库集合**：创建必要的集合
2. **优化云函数超时处理**：增加超时设置和重试机制
3. **完善错误处理**：添加更详细的错误信息
4. **文本审核功能**：新增文本内容审核功能
5. **修复图片上传流程**：先上传到云存储，再审核云存储中的图片

## 功能说明

### 图片审核功能
- 自动审核书籍封面图片
- 支持重试机制（最多重试2次）
- 超时时间：5秒

### 文本审核功能
- 自动审核书籍标题、作者、出版社、课程、描述等文本内容
- 支持多种违规类型检测：色情、谩骂、广告、垃圾、政治敏感、恐怖主义等
- 审核失败会显示具体的违规字段和原因
- 超时时间：10秒

## 操作步骤

### 1. 初始化数据库集合
1. 在小程序中进入"我的"页面
2. 在"我的"页面中找到并点击"数据库初始化"页面
3. 点击"初始化数据库"按钮
4. 等待初始化完成，查看结果

### 2. 上传云函数（必须按顺序上传）
需要上传以下云函数（请逐个上传）：
1. `checkImage` - 图片审核云函数（已修复ENOTFOUND tmp错误）
2. `checkText` - 文本审核云函数
3. `publishBook` - 发布书籍云函数（已优化错误处理）
4. `initDatabase` - 数据库初始化云函数

**重要提示**：请先上传checkImage云函数，修复图片审核功能！

### 3. 测试发布书籍功能
1. 进入"发布书籍"页面
2. 填写书籍信息并上传图片
3. 点击"发布"按钮
4. 检查是否发布成功

### 4. 测试文本审核功能
1. 尝试输入包含敏感词的书籍标题、描述等
2. 点击"发布"按钮
3. 查看是否正确拦截并提示违规内容

## 注意事项
1. **首次初始化**：如果这是第一次初始化，系统会创建所有必要的数据库集合
2. **云函数上传**：每次修改云函数后需要重新上传
3. **超时处理**：图片审核现在有重试机制，最多重试2次
4. **错误信息**：详细的错误信息会显示在控制台和日志中
5. **文本审核**：文本审核会检查所有文本字段，包括标题、作者、出版社、课程、描述
6. **敏感词**：系统会自动检测色情、谩骂、广告、垃圾、政治敏感、恐怖主义等违规内容
7. **人工复审**：部分审核可能需要人工复审，书籍会标记为待审核状态
8. **图片上传流程**：修复后的流程是：先上传到云存储 → 再审核云存储中的图片 → 审核通过后才能使用
9. **违规图片处理**：如果图片审核不通过，会自动删除已上传的图片

## 常见问题

### Q: 出现"getaddrinfo ENOTFOUND tmp"错误怎么办？
A: 这是图片审核功能的一个已知问题，已经修复。请按以下步骤操作：
1. 重新上传`checkImage`云函数
2. 如果仍然出现错误，请重启微信开发者工具
3. 清除小程序缓存后重试

### Q: 图片审核失败后图片会怎样？
A: 图片审核不通过的图片会自动从云存储中删除，不会占用您的存储空间。

### Q: 上传图片时提示"图片路径无效"怎么办？
A: 这通常是因为上传过程中出现异常导致的，请重新选择图片并上传。

### Q: 为什么需要先上传到云存储再审核？
A: 之前的做法是直接审核临时文件路径，但云端无法访问本地临时文件，导致审核失败。修复后的流程更加稳定可靠。

如果仍然出现问题，请检查：
1. 云函数是否正确上传
2. 数据库权限是否正确配置
3. 网络连接是否正常
4. 图片大小是否超过限制（5MB）
